# catchtable-sniper

캐치테이블에서 원하는 식당의 빈자리가 나오면 자동으로 예약을 잡아주는 스킬.

## 트리거 조건

다음 중 하나에 해당하면 이 스킬을 사용한다:
- "캐치테이블 [식당명] 빈자리 나오면 예약해줘"
- "캐치테이블 [식당URL] [날짜] [인원] 자동예약"
- "캐치테이블 스나이핑"
- "[식당명] 예약 잡아줘 캐치테이블"

## 입력 파싱

사용자 입력에서 다음을 추출한다:

| 항목 | 예시 | 필수 여부 |
|------|------|----------|
| 식당명 또는 URL | "온지음" / `app.catchtable.co.kr/ct/shop/onjium` | 필수 |
| 날짜 | "5월 3일", "이번 주 토요일" | 필수 |
| 인원 | "2명", "4인" | 필수 |
| 시간대 | "저녁", "19시 이후" | 선택 (없으면 전체) |
| 폴링 간격 | "30초마다" | 선택 (기본: 30초) |

## 실행 플로우

### STEP 1 — 브라우저 준비 및 로그인 확인

Chrome MCP로 캐치테이블 앱에 접속한다.

```
navigate: https://app.catchtable.co.kr
```

로그인 상태 확인:
- 우측 하단 "MY" 탭 클릭
- 로그인된 경우: 닉네임/프로필 표시 → STEP 2로
- 로그인 안 된 경우:
  ```
  사용자에게 안내:
  "캐치테이블에 로그인되어 있지 않습니다.
   Chrome에서 캐치테이블에 카카오/네이버 로그인 후 다시 실행해주세요."
  ```
  → 스킬 중단 (로그인 자동화는 하지 않음)

### STEP 2 — 식당 페이지 탐색

식당명이 주어진 경우 검색:
```
navigate: https://app.catchtable.co.kr/ct/search?keyword={식당명}&type=shop
```

검색 결과에서 정확한 식당 링크(`/ct/shop/` 포함) 추출 후:
```
navigate: https://app.catchtable.co.kr/ct/shop/{shopUrl}
```

식당 페이지가 로드되면 `shopSeq` 값을 캡처한다:
- 페이지 소스 또는 네트워크 요청에서 `shopSeq` 추출
- 또는 URL에서 shop slug 사용

### STEP 3 — 예약 가능 일정 확인

식당 페이지에서 "예약하기" 버튼 클릭 → 날짜 선택 UI 진입

**방법 A (직접 API 폴링 — 로그인 쿠키 있을 때):**
```
GET https://ct-api.catchtable.co.kr/api/v3/schedule/onlineOpenSchedule/main
  ?shopSeq={shopSeq}&year={YYYY}&month={MM}
```

**방법 B (UI 폴링 — 항상 동작):**
1. 날짜 선택 캘린더에서 목표 날짜로 이동
2. 해당 날짜 클릭
3. 시간 슬롯 목록 파싱
4. 빈 슬롯(활성화된 버튼) 존재 여부 확인

### STEP 4 — 폴링 루프

빈자리가 없으면 다음을 반복한다:

```
while 빈자리 없음:
    {폴링 간격}초 대기
    페이지 새로고침 또는 해당 날짜 재클릭
    슬롯 확인
    빈자리 발견 시 → STEP 5로 즉시 이동
```

폴링 간격 기본값: 30초
최대 폴링 시간: 2시간 (초과 시 사용자에게 알림 후 중단)

중간 상태 출력 형식:
```
[14:23:15] 5월 3일 저녁 슬롯 확인 중... 빈자리 없음 (30초 후 재시도)
[14:23:45] 5월 3일 저녁 슬롯 확인 중... 빈자리 없음
[14:24:15] ✅ 빈자리 발견! 19:30 (2인) — 예약 시작
```

### STEP 5 — 자동 예약 진행

빈 시간 슬롯 버튼 즉시 클릭 → 예약 폼 진입

폼 자동 입력:
- 인원수: 사용자가 지정한 인원 선택
- 방문 목적: "식사" (기본값)
- 주의사항 동의: 전체 동의 체크
- 예약자 정보: 앱에 저장된 정보 자동 사용 (별도 입력 불필요)

**결제가 필요한 경우 (선결제 식당):**
```
사용자에게 알림:
"빈자리를 발견했습니다! 결제가 필요합니다.
 결제 금액: {금액}원
 지금 결제를 진행할까요? (예/아니오)"
```
→ 사용자 확인 후 결제 진행 (결제 정보 자동 입력은 하지 않음)

**무료 예약인 경우:**
"예약하기" 최종 확인 버튼 클릭

### STEP 6 — 완료 확인

예약 완료 화면 캡처 후 사용자에게 보고:

```
🎉 예약 완료!

식당: {식당명}
날짜: {날짜}
시간: {시간}
인원: {인원}명
예약번호: {예약번호}

캐치테이블 앱 > MY > 예약내역에서 확인 가능합니다.
```

## 에러 핸들링

| 상황 | 대응 |
|------|------|
| 식당 페이지 404 | "식당을 찾을 수 없습니다. 이름을 다시 확인해주세요." |
| 예약 오픈 전 | 오픈 일정 안내 후 오픈 시간에 맞춰 재시작 |
| 슬롯 클릭 후 이미 마감 | 즉시 재폴링 재개 |
| 네트워크 오류 | 10초 후 재시도, 3회 연속 실패 시 사용자 알림 |
| 2시간 초과 | "2시간 동안 빈자리가 없었습니다. 계속 시도할까요?" |

## 주의사항

- 이 스킬은 로그인된 Chrome 브라우저 세션을 사용합니다.
- 선결제 식당의 결제 정보는 직접 입력해야 합니다.
- 과도한 폴링은 서버에 부하를 줄 수 있으므로 최소 30초 간격을 유지합니다.
- 캐치테이블 이용약관을 준수하는 범위에서 사용하세요.

## 사용 예시

```
"온지음 5월 10일 저녁 2인 빈자리 나오면 예약해줘"
"https://app.catchtable.co.kr/ct/shop/mingles 토요일 4명 자동예약"
"스시 오마카세 다음주 주말 2인 캐치테이블 스나이핑"
```
