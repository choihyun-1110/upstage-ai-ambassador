## 어떻게 동작하나

이 스킬은 **Claude Code Skill** 형태로 만들었다. Claude Code에 불러오면 채용공고 URL이나 PDF를 주는 것만으로 자동으로 동작한다.

전체 흐름은 이렇다:

```
채용 사이트 URL 입력
 ↓
Chrome MCP → 페이지 접속 + JS 렌더링까지 대기
 ↓
목록 페이지라면: 전체 공고 링크 자동 추출 → 각 공고 순차 방문
 ↓
각 JD 텍스트 수집
 ↓
Upstage Solar Pro → 사용자 프로필과 비교 분석
 ↓
적합도 점수(0-100) + 강점/약점/추천 이유 출력
 ↓
여러 공고라면 랭킹 테이블로 정리
```

사용자 입장에선 딱 두 단계다.

1. URL 던지기 (또는 PDF 경로)
2. 결과 받기

## 설계에서 제일 많이 고민한 것

**"JS 렌더링 사이트를 어떻게 읽을 것인가"**

기아, 삼성, 현대 같은 대기업 채용 사이트는 전부 SPA(Single Page Application)다. 일반 HTTP 요청으로 URL을 긁으면 빈 껍데기 HTML만 나온다. 콘텐츠가 JavaScript로 로딩되기 때문이다.

처음에 `WebFetch`로 시도했다가 이걸 깨달았다. 그래서 Chrome MCP를 사용했다. Chrome MCP는 실제 브라우저 엔진이 렌더링한 결과를 읽을 수 있어서 JS 렌더링 사이트도 문제가 없다.

```
# 이 방식은 안 됨 (빈 HTML)
WebFetch → career.kia.com

# 이 방식이 됨 (실제 콘텐츠)
Chrome MCP navigate → get_page_text
```

**"Solar Pro 프롬프트 설계"**

단순히 "이 JD랑 이 프로필 비교해줘"로는 결과가 들쭉날쭉했다. 점수 기준도 없고, 출력 형식도 매번 달랐다.

그래서 분석 구조를 명확히 지정했다.

```
점수 기준: 필수 요건 충족 여부(40%) + 우대 조건 일치도(30%) + 커리어 방향성(30%)
출력: 점수 / 강점 bullet / 약점+보완방법 bullet / 추천여부+이유 / 강조할 포인트 3가지 / 키워드
```

이렇게 하니 6개 공고의 결과물이 일관된 형식으로 나와서 랭킹 테이블을 만들기도 쉬웠다.

## 만들면서 힘들었던 것

**kia-autoworld.com의 모달 방식**

랜딩 페이지의 "자세히 보기" 버튼들이 전부 `href="#"`이었다. 클릭하면 JS 모달이 뜨고, 거기서 각 직무 링크가 나오는 구조다.

처음엔 DOM에서 링크를 바로 추출하려고 했는데 당연히 `href="#"`만 보였다. 결국 JavaScript로 버튼을 직접 클릭하고, 모달이 열린 뒤 그 안의 링크를 다시 추출하는 방식으로 해결했다.

```javascript
// 직접 클릭으로 모달 열기
document.querySelectorAll('.item-link.pop_view')[1].click();

// 모달 안에서 링크 추출
[...document.querySelectorAll('a[href*="applyView"]')]
  .map(a => a.pathname)
```

**career.kia.com 무한스크롤**

채용 목록 페이지가 10개씩 끊어서 로딩되는 방식이었다. 전체 44건 중 IT 신입은 7페이지에 흩어져 있었다. 페이지 전체를 다 긁으려고 스크롤 이벤트도 트리거해보고 여러 시도를 했는데, 결국 페이지 전역 함수 `getList(page)`를 발견해서 직접 호출하는 방식으로 전체 데이터를 불러왔다.

```javascript
// 페이지에 선언된 전역 함수로 전체 페이지 로드
for (let p = 2; p <= listTotal; p++) {
  getList(p);
}
```

이런 걸 찾아내는 과정이 생각보다 재밌었다. 사이트마다 방식이 다 달라서 매번 탐정처럼 파고들어야 한다.

## 왜 Claude Code Skill인가

n8n 워크플로우와 비교했을 때 Claude Code Skill의 장점이 뚜렷하게 있었다.

| 비교 | n8n 워크플로우 | Claude Code Skill |
|------|---------------|-------------------|
| JS 렌더링 사이트 대응 | ❌ 불가 | ✅ Chrome MCP |
| 설치 난이도 | 중간 (n8n 서버 필요) | 낮음 (파일 복사 1개) |
| 유연성 | 정해진 노드 내에서만 | 동적 대응 가능 |
| 결과 형식 | 고정 | 상황에 맞게 조정 |

특히 "JS 렌더링 사이트 자동 크롤링"은 n8n으로는 구현이 어렵거나 복잡한 우회가 필요한데, Claude Code + Chrome MCP 조합이면 자연스럽게 해결된다.
